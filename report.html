2. Incorrect Specification — Verification Failure

To test Dafny’s verification mechanism, the correct postcondition was intentionally replaced with an incorrect one. Dafny correctly rejected this version. 

// Incorrect version (used only for testing)
method Abs(x: int) returns (x': int)
  ensures x' >= 0
  ensures x' == x   // Incorrect postcondition
{
  if x < 0 {
    x' := -x;
  } else {
    x' := x;
  }
}

Dafny reports a verification failure because the method returns -x for negative inputs, which does not satisfy the incorrect postcondition x' == x.

Figure: Verification failure due to incorrect postcondition.
